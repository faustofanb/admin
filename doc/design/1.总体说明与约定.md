# 第 1 章 总体说明与约定（Detailed Design）

> 文档编号：DD-01
> 版本：v1.0
> 更新时间：2025-11-20
> 适用项目：多租户后台管理系统（Java 21 + Spring Boot + Vben5）

---

## 1.1 设计目标

本系统旨在构建一个 **高性能、高可用、可扩展的多租户后台管理平台**，用于统一管理：

- 租户
- 用户
- 组织（机构/部门）
- 菜单
- 角色与权限
- 资源（文件等）
- 日志
- 定时/异步任务
- Excel 批处理
- 复杂业务流程（可选 LiteFlow）

在总体设计上，满足以下几个核心目标：

1. **多租户隔离**
   - 所有核心数据和访问控制都必须明确带上 `tenantId`。
   - 任何一个租户的操作不得干扰其它租户的数据和行为。
2. **安全可控**
   - 鉴权必须细粒度（用户—角色—权限—资源）。
   - 传输加密、审计完备、操作可追踪。
3. **高并发与高性能**
   - 利用 Java 21 虚拟线程处理 IO 密集场景，实现“少连接高并发”。
   - 通过 Redis/Redisson、Sentinel、缓存与限流策略缓解热点压力。
4. **可观测与易运维**
   - PLG（Promtail + Loki + Grafana）实现日志集中化查询。
   - Micrometer + Prometheus + OpenTelemetry 实现指标与 Trace 全链路观测。
   - 所有关键业务流至少具备：日志、指标、Trace 三种观测手段。
5. **可演化与可拆分**
   - 当前采用 Spring Boot 单体多模块。
   - 保留跨模块接口边界（Dubbo rpc-api），以及异步事件边界（Outbox + RocketMQ），为未来微服务拆分预留路径。
6. **TDD 驱动开发**
   - 领域逻辑、应用服务和关键基础设施接口以测试驱动开发。
   - 尽量在领域层和应用层先通过单元/组件测试，再进行集成与端到端测试。

## 1.2 背景与约束条件

### 1.2.1 业务背景

- 系统要服务于多个不同租户（企业/组织），每个租户有自己的：
  - 用户体系
  - 组织架构
  - 菜单配置
  - 角色与权限规则
  - 资源（文件/文档等）
- 共性需求：
  - 各租户之间 **逻辑隔离**（默认同库同表、带 tenantId）。
  - 支持 **混合隔离模式**：关键租户可配置独立 Schema 或 Database（通过动态数据源路由）。
  - 后端统一一套代码，多租户共享资源与基础设施。

### 1.2.2 技术背景

- 引入 Java 21 虚拟线程（Project Loom），配合 Spring Boot 3.x，要求：
  - 尽量用虚拟线程承载所有 IO 密集型操作（数据库、Redis、MQ、RustFS 等）。
  - 注意与 ThreadLocal 依赖框架（Seata、Security、Dubbo）兼容性。
- 目标生态组合：
  - **后端**：Java 21 + Spring Boot 3.x + Maven 多模块
  - **ORM**：Jimmer
  - **安全**：Spring Security + JWT
  - **中间件**：Redis/Redisson、RocketMQ、PostgreSQL、BlobStorage (S3/MinIO/RustFS)、PowerJob、Sentinel、Seata、Spring Cloud Gateway
  - **可观测**：PLG、Prometheus/Micrometer、OpenTelemetry
  - **前端**：Vben5（Vue 3 + TypeScript）

### 1.2.3 非功能性约束

- 初始预期部署环境：
  - Kubernetes / Docker 化部署，具备容器编排能力。
  - 至少 2~3 副本支持水平扩展。
- 时延目标（初版参考值，后续在性能测试中固化）：
  - 用户查询：P99 < 120ms
  - 资源上传（中小文件 < 10MB）：P95 < 800ms（包括 RustFS 写入）
  - Excel 导入：10k 行，整体流程 < 6 秒（完成入库）

## 1.3 总体架构概览

### 1.3.1 架构风格与分层

本系统采用 **分层架构 + DDD + CQRS + 事件驱动 +（可选）流程编排**：

- **表示层（Interface Layer）**
  - HTTP REST API / WebSocket / SSE（对前端 Vben5）
  - Dubbo RPC Endpoint（对内部或未来微服务）
- **网关层（Gateway Layer，基于 Spring Cloud Gateway）**
  - 统一鉴权、限流、路由转发。
  - 负责 SSE/WebSocket 连接维护与消息推送。
- **应用层（Application Layer）**
  - Command Handler：处理写操作（Create/Update/Delete），负责用例编排。
  - Query Handler：处理读操作，支持分页、过滤等。
- **领域层（Domain Layer）**
  - 聚合（Aggregate）、实体（Entity）、值对象（Value Object）
  - 领域服务（Domain Service）
  - 领域事件 （Domain Event）
- **基础设施层（Infrastructure Layer）**
  - 数据访问：Jimmer + PostgreSQL
  - 缓存：Redis + Redisson
  - 消息：RocketMQ
  - 任务调度：PowerJob Worker
  - 文件存储：RustFS 客户端
  - 限流熔断：Sentinel
  - 分布式事务：Seata（准备/按需）
  - 可观测：Micrometer/Prometheus，Promtail/Loki, OpenTelemetry
  - RPC：Dubbo（当前 injvm，将支持远程）
- **适配与集成层（Adapter Layer）**
  - RPC 契约模块（rpc-api）
  - RPC 实现模块（rpc-impl）
  - 中间件集成模块（infra-integration）
  - 事务与一致性支持模块（transaction-support）
  - （可选）flow-engine：LiteFlow 编排引擎封装

### 1.3.2 逻辑架构图（文字版）

```text
[Vben5 前端]
    │  REST/JSON
    ▼
[Controller 层]
    │ 调用
    ▼
[Application 层]
  ├─ Command Handler (写)
  └─ Query Handler (读)
    │ 依赖
    ▼
[Domain 层]
  ├─ 聚合/实体/值对象
  ├─ 领域服务
  └─ 领域事件发布
    │ 依赖
    ▼
[Infrastructure 层]
  ├─ Jimmer + PostgreSQL
  ├─ Redis + Redisson
  ├─ RocketMQ
  ├─ RustFS Client
  ├─ PowerJob Worker
  ├─ Sentinel
  ├─ Seata
  └─ 可观测组件 (PLG + Prometheus + OTel)
```

## 1.4 技术栈与选型理由

### 1.4.1 后端技术栈

| 领域             | 选型                      | 理由简述                                            |
| ---------------- | ------------------------- | --------------------------------------------------- |
| 语言             | Java 21                   | 虚拟线程、主流生态、工具链成熟                      |
| 框架             | Spring Boot 3.x           | 微服务生态、与 Spring Security 等集成好             |
| ORM              | Jimmer                    | 类型安全、自动生成实体、双向关联友好、简化 DDD 落地 |
| 安全             | Spring Security           | 成熟的认证鉴权框架，与 JWT/OAuth2 易集成            |
| 缓存客户端       | Redisson                  | 提供丰富的分布式结构（锁、限流器、信号量、队列等）  |
| 任务调度         | PowerJob                  | 分布式调度、可视化管理、支持 Java SDK               |
| 消息队列         | RocketMQ                  | 成熟可靠、有序消息、延时消息等支持良好              |
| 分布式事务       | Seata                     | AT/SAGA 模式支持、与 Spring 生态集成                |
| 限流熔断         | Sentinel                  | 丰富规则类型、支持热点参数限流                      |
| 资源存储         | RustFS                    | 高性能文件存储，适合 IO 密集型资源服务              |
| Excel            | EasyExcel                 | 流式读写、大文件处理效率高                          |
| 可观测           | Micrometer + Prometheus   | 指标采集与告警主流组合                              |
| 日志平台         | Promtail + Loki + Grafana | 统一日志采集与可视化                                |
| Trace            | OpenTelemetry             | 标准化链路追踪，跨组件协作                          |
| RPC              | Dubbo                     | Java 领域成熟 RPC 框架，支持多协议与未来演化        |
| 流程编排（可选） | LiteFlow                  | 轻量级、DSL 风格流程引擎，适合复杂可配置业务流      |

### 1.4.2 前端技术栈

- Vben Admin 5（基于 Vue 3 + TypeScript）：
  - 内置完善的后台管理 UI 组件与权限路由方案。
  - 提供 Vue 3 Composition API + TS 强类型开发模式。
  - 支持主题、菜单、表单等高度定制。

## 1.5 模块与包结构约定

### 1.5.1 Maven 模块划分

建议 Maven 结构（示意）：

```text
pom.xml (parent)
 ├─ backend-common/
 ├─ backend-core/
 ├─ backend-infra/
 ├─ backend-transaction/
 ├─ backend-rpc-api/
 ├─ backend-rpc-impl/
 ├─ backend-gateway/
 ├─ backend-boot/
 └─ module/
     ├─ backend-system/
     ├─ backend-file/
     ├─ backend-batch/
     ├─ backend-schedule/
     └─ backend-audit/
```

### 1.5.2 包结构风格（以 backend-system 为例）

```text
io.github.faustofanb.admin.system
  ├─ app
  │   ├─ command
  │   └─ query
  ├─ domain
  │   ├─ model
  │   ├─ service
  │   └─ event
  ├─ infra
  │   ├─ repository
  │   └─ mapper (Jimmer 生成或适配)
  ├─ adapter
  │   ├─ rest
  │   └─ rpc
  └─ config
```

- **app**：应用服务与命令/查询处理逻辑。
- **domain**：领域模型与规则。
- **infra**：与数据库、缓存等外部系统交互的实现。
- **adapter**：对外接口（REST、RPC）的适配层。

## 1.6 统一约定

### 1.6.1 命名约定

- DTO：`*DTO`（用于对外返回和 RPC 契约）
- VO：`*VO`（视图对象，用于前端展示）
- Command：`*Command`
- Query：`*Query`
- 聚合根：单数名词（如 `User`, `Tenant`）
- 领域事件：`*Event`

### 1.6.2 错误码约定

| 范围     | 前缀  | 说明                       |
| -------- | ----- | -------------------------- |
| 认证     | AUTH  | 登录、Token 相关           |
| 授权     | PERM  | 权限不足、租户越权         |
| 参数校验 | VALID | 校验错误                   |
| 业务冲突 | BIZ   | 状态不允许、并发冲突       |
| 限流     | RATE  | Sentinel/Redisson 限流触发 |
| 系统     | SYS   | 未捕获异常、依赖不可用等   |

示例返回结构：

```json
{
  "code": "AUTH_INVALID_TOKEN",
  "msg": "访问令牌无效或已过期",
  "traceId": "f5c3...",
  "data": null
}
```

### 1.6.3 时间与时区

- 数据库统一存储为 UTC 时间。
- API 返回 ISO 8601 字符串，前端在本地转换为用户时区显示。
